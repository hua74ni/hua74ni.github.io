ztree异步加载

  • 由于ztree的数据量增大，通过直接将所有的ztree的节点都全部加载的方式，就会变得越来越慢的，所以为了解决这个问题。

    解决的方案：
    每次点击parent节点时才去加载此下的子节点挂于此节点下，点击树节点，table/grid回显示对应的数据，同时得注意：

    1. 此parent节点下异步加载的子节点需要判断是否是父节点，否则无法显示是否parent也就是无打开节点的按钮。
    2. 由于需要判断是否是父节点，所以需要后台传递值给前端带有isParent(true|false)。
    3. 如果没有isParent这个字段也会影响，zTreeObj.addNodes这个方法无法有效果。
    4. 同时为了实现增删改节点时，保持原本节点打开状态，而不是重新加载并且完全闭合。

update

add

前端提交新增保存完controller后，返回对应的新增的节点(json)，ajax的callback方法中对前端的ztree新增显示节点

delete

前端提交批量删除保存完controller后，返回操作成功提示，ajax的callback方法中对前端的ztree进行批量删除

move

树节点进行移动.
